## 已完成修复

### P0-1 标准流水线 baseline 无法跑通

- 修复 `engine/descriptive.py`：默认自动选择数值列，并调用 `stats_core.run_descriptive(df, numeric_cols)`。
- 修复 `engine/correlation.py`：默认自动选择数值列，并调用 `stats_core.run_correlation(df, numeric_cols)`。
- 新增回归测试：`tests/regression_p0_p1.py`

验收结果：

- `python engine/descriptive.py tests/fixtures/panel_data.parquet`：退出码 0，JSON 可解析。
- `python engine/correlation.py tests/fixtures/panel_data.parquet`：退出码 0，JSON 可解析。

### P1-1 数据格式承诺与实现不一致

- 新增 `engine/lib/cli_utils.py`。
- 新增 `load_dataframe()`，支持 `.csv`、`.xlsx/.xls`、`.parquet`、`.dta`。
- `data_audit.py` 改为使用统一 loader。

验收结果：

- `python engine/data_audit.py tests/fixtures/panel_data.csv --method panel`：退出码 0，JSON 可解析。

### P1-2 Panel 数据识别规则过窄

- `data_audit.py` 新增 entity 候选：`entity/id/firm_id/code/company_id/stock_code`。
- 新增 time 候选：`time/year/date/period`。
- `method_match.detected` 会写出识别到的 entity/time 列。

验收结果：

- `tests/fixtures/panel_data.csv` 和 `tests/fixtures/panel_data.parquet` 均识别为 `entity=code`、`time=year`。

### P1-3 非法 JSON config 暴露 traceback

- 新增 `parse_json_config()`，统一处理普通 JSON 和 PowerShell 转义 JSON。
- 已覆盖全部 `engine/*.py` 中的 `json.loads(sys.argv[2])` 入口。
- 非法 JSON 统一输出 `{error: "Invalid JSON config: ..."}`，退出码 1。

验收结果：

- `python engine/ols.py tests/fixtures/panel_data.parquet "{bad json}"`：退出码 1，stdout 为 JSON，stderr 无 traceback。

### P1-4 报告协议仍要求 Tab 分隔复制块

- `protocols/reporting.md` 改为默认不输出 Tab 分隔复制块。
- `templates/paper.md` 改为默认不附 Tab 分隔复制块。
- 仅用户明确要求可复制表格时，才额外提供 TSV。

### P1-5 requirements 未声明 panel 必需依赖

- `requirements.txt` 已加入 `linearmodels>=5.0`。

### P1-6 禁止 LLM 跳过 engine 的行为测试

- `protocols/anti_hallucination.md` 新增“绕过引擎型”禁止行为。
- 新增 `tests/llm_behavior_redteam.md`，覆盖诱导手写 OLS、诱导 RD 近似替代、诱导编造预期结果。

### P1-7 回归表期刊格式动态验收

- 新增 `tests/regression_table_format_p1.md`。
- 明确回归表必须包含：期刊表号、混合 OLS + FE 列、系数星号、括号 t 值、N/R²/F/Hausman/标准误脚注、可信度标签。
