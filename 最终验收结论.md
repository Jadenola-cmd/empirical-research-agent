# Empirical Research Agent Skill 最终验收结论

## 结论

P0/P1 修复后，当前版本已具备进入下一轮全量验收的条件；是否进入全局 skill 库，建议以 P2 清理和人工 LLM 行为红队测试结果作为最终准入依据。

此前阻断标准流水线的 baseline 问题已修复：`descriptive.py` 与 `correlation.py` 均可对 `test-data/panel_data.parquet` 正常输出 JSON。

## 已通过能力

- Skill 总入口具备清晰路由：标准流水线、代码生成、结果解读、方法咨询、探索模式均有规则。
- 协议层具备关键护栏：反幻觉、方法边界、变量确认、可信度标签均有明确要求。
- OLS engine 可用，修正参数后能输出合法 JSON。
- Panel FE engine 可用，修正参数后能输出合法 JSON，并包含 Hausman 字段。
- 多数负向业务错误能返回结构化 JSON。
- 历史 v2 HTML 报告已体现变量定义表、混合 OLS + FE/RE、Hausman、聚类标准误和可信度标签。

## 剩余待观察能力

- LLM 行为红队测试用例已补充，但仍需在真实 Agent 会话中人工验收。
- 回归表期刊格式验收用例已补充，但仍需在真实报告生成链路中执行。
- P2 易用性问题仍可继续优化，例如 PowerShell `--config-file` 支持、中文 notes 编码治理。

## 修复后回归清单

修复后至少重跑以下命令：

```powershell
python engine/data_audit.py test-data/panel_data.parquet --method panel
python engine/descriptive.py test-data/panel_data.parquet
python engine/correlation.py test-data/panel_data.parquet
python engine/ols.py test-data/panel_data.parquet '{\"dep_var\":\"LEV\",\"indep_vars\":[\"ROA\"],\"control_vars\":[\"GROWTH\",\"SIZE\"],\"robust_se\":true}'
python engine/panel.py test-data/panel_data.parquet '{\"dep_var\":\"LEV\",\"indep_vars\":[\"ROA\"],\"control_vars\":[\"GROWTH\",\"SIZE\"],\"entity_var\":\"code\",\"time_var\":\"year\",\"model_type\":\"fe\",\"cluster_var\":\"code\"}'
```

同时重跑负向测试：

- 不存在的数据路径
- CSV 直接传给 `data_audit.py`
- 非法 JSON config
- 变量名不存在
- PSM 非二值 treatment
- PSM treated/control 样本不足
- DID 缺少 time 参数

## 最小准入标准

- P0 问题清零。
- 核心 engine 冒烟测试全部退出码为 0，且输出可解析 JSON。
- 所有失败场景返回结构化 JSON 错误，不能暴露 traceback。
- 标准样例可以完成：数据质检、描述统计、相关矩阵、混合 OLS、Panel FE、报告输出。
- 报告协议不再默认要求 Tab 分隔复制块。
