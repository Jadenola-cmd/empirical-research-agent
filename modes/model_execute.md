# Model Execute Mode — 模型执行模式

## 定位

模型执行模式用于用户已经给出最低可执行 `model_spec`，并明确要求跑模型或生成回归报告的场景。Agent 只按用户设定执行，不扩展研究问题，不替用户重定义变量方向。

## 进入条件

用户已明确给出最低可执行 `model_spec`：

- 因变量 Y。
- 核心自变量 X。
- 方法类型，例如 OLS、面板 FE/RE、DID、PSM、调节效应、中介效应。
- 执行意图，例如“跑模型”“做回归”“生成回归报告”。

控制变量、固定效应、聚类或稳健标准误、样本筛选或合并方式可以暂缺。暂缺时仍可进入模型执行模式，但必须在 Step 4A/4B 阻断确认，不能由 Agent 自动补全。

## 执行规则

- 严格按用户设定执行。
- 不得擅自更换 Y/X。
- 不得重定义变量方向。
- 不得扩展新的研究问题。
- 若发现设定缺失或冲突，先回到 Step 4A/4B 阻断确认。
- 用户只给主题或模糊问题、未达到最低可执行 `model_spec` 时，转入研究探索模式。

## 输出边界

允许输出：

- 按用户设定生成的描述统计、相关性和模型结果。
- 对 engine 输出的可信度标注。
- 论文式三线表和解读。

不得输出：

- 未经用户授权的替代模型作为主结论。
- 未经确认的变量构造或样本合并结果。
