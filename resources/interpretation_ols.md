# 解读检查清单：OLS回归

适用于 `run_analysis.py` 的 `ols` 方法输出。

## 1. 系数符号与方向是否符合理论预期
- 先核对核心解释变量（`indep_vars`）系数符号是否与论文假设方向一致；若不一致，不要直接删除该变量重跑，应在讨论部分正面回应（如"与预期相反，可能的解释是……"），删除不显著或反向变量去"凑"显著结果是典型的p-hacking，审稿人会重点关注这一点。
- 虚拟变量系数（输出中 `categorical_vars`/`dummy_vars` 标注的列）的符号要对照被省略的基准类别（`drop_first=True` 省略的那一类）解读，写作时必须说明基准类别是谁，否则读者无法判断方向含义。

## 2. 显著性阈值如何表述
- 严格按输出中的 `sig` 字段（`***`/`**`/`*`/空）配合 `p_value` 表述，规范写法："X对Y的影响在1%水平上显著为正（β=0.xx, p<0.01）"，不要只写"显著"不带水平，也不要把p=0.08说成"显著"而不注明是10%水平。
- 若核心变量p值在0.1附近（如0.08-0.12），属于边缘显著，论文中应如实说明"边缘显著"而非夸大为"显著"或回避不提，审稿人复核时容易发现夸大表述。

## 3. 拟合优度的合理区间与异常提示
- 截面数据R²通常在0.1-0.5区间属正常；若 `r2`/`r2_adj` 超过0.9，提示存在严重多重共线性或解释变量中混入了被解释变量的同源信息（如用同一指标的子项做自变量），应先排查再写作，不要把高R²当作"模型拟合很好"的证据直接写进论文。
- 关注输出中的 `f_stat`/`f_pvalue`：F检验不显著（f_pvalue较大）说明整个模型的解释力存疑，即使个别系数显著也需谨慎解读，建议在论文脚注中报告F统计量。

## 4. 标准误类型是否说明清楚
- 输出的 `se_type` 字段（`conventional`/`robust(HC1)`/`clustered(var)`）必须在论文表格脚注中明确写出，例如"括号内为稳健标准误（Robust HC1）"；混用不同SE类型的模型放在同一张对比表时尤其要逐列标注，否则读者无法判断显著性结论是否受SE选择驱动。
- 若数据存在异方差迹象（如残差图呈喇叭状）或分组聚类结构（如同一公司多年观测），应使用 `robust_se=True` 或 `cluster_var` 聚类到对应分组，而非默认的普通标准误——普通SE在这些场景下容易低估标准误、虚增显著性。

## 5. 审稿人常见追问点
- "解释变量之间是否存在多重共线性？"——本工具未输出VIF，若R²异常高或系数符号与单变量回归差异很大，建议补充说明已检查过相关系数矩阵（参见 `interpretation_descriptive.md`）排除严重共线性。
- "为什么不用面板固定效应模型？"——若数据本身是面板结构（有个体+时间维度）却只跑了OLS，审稿人几乎必问。应说明理由（如非面板数据，或面板固定效应不适用），或改用 `panel` 方法重新估计。
- "控制变量的选择依据是什么？"——`control_vars` 的纳入应有理论依据（遗漏变量偏误的考虑），不能为了让核心变量显著而临时增删控制变量，论文方法部分需说明控制变量选取的理论依据。
