# Result Validation Protocol — 结果验证与可信度评级

## 位置

后置阻断 [P3]：在 protocol_order.md 中位于 workflow 执行之后、reporting 之前。

## 可信度标签体系

每个分析结论必须标注以下三种标签之一：

### [VERIFIED] — 可验证

**条件**（同时满足）：
- 数字直接来自 engine/ 脚本输出
- 方法满足所有前置条件（data_audit 通过）
- 统计检验通过（如 Hausman 有效、VIF < 10、异方差检验不显著）

**允许用途**：可直接进入论文正文建议

**示例**：
```markdown
leverage 对 roa 的回归系数为 -0.532（p<0.01），表明资产负债率对盈利能力有显著负向影响。
[VERIFIED] 来源: engine/ols.py 输出, VIF<5, 异方差检验通过
```

### [INFERRED] — 可推断

**条件**（满足任一）：
- 数字来自 engine 输出，但存在方法论上的注意事项
  - 如 Hausman 检验结果为负（保守选择FE）
  - 如样本量偏小（N < 100）
  - 如 R² 偏低（< 0.1）
- 多个 [VERIFIED] 结论的综合推论（逻辑链清晰）

**允许用途**：可进入论文讨论部分，但需注明推断依据和局限性

**示例**：
```markdown
综合列(1)-(3)的结果，公司规模对盈利能力的正向效应在控制杠杆率后仍然稳健。
[INFERRED] 基于三组OLS回归的系数符号一致性推断，但未做正式的中介效应检验
```

### [SPECULATIVE] — 推测性

**条件**（满足任一）：
- Agent 基于领域知识的推测，无 engine 输出直接支撑
- 数据存在明显质量问题但用户要求继续
- 方法假设不完全满足但用户坚持使用
- 结论外推到样本外情境

**禁止用途**：不得进入论文正文或结论建议

**允许用途**：可作为"未来研究方向"或"探索性发现"供用户参考

**示例**：
```markdown
杠杆率的负向效应可能在经济下行期更为显著，但当前样本未覆盖完整经济周期。
[SPECULATIVE] 基于财务理论推测，未做经济周期分组检验
```

## 评级流程

```
engine 输出
  │
  ├─ 数字是否来自 engine？
  │   └─ 否 → [SPECULATIVE]（违反 anti_hallucination 协议，应终止）
  │
  ├─ data_audit 是否通过？
  │   └─ 否 → 降级（PASS→可[VERIFIED], WARN→最高[INFERRED], BLOCK→最高[SPECULATIVE]）
  │
  ├─ 方法假设是否满足？
  │   ├─ 全部满足 → 可评 [VERIFIED]
  │   ├─ 部分满足 → 最高 [INFERRED]
  │   └─ 不满足 → 最高 [SPECULATIVE]
  │
  ├─ 统计检验是否通过？
  │   ├─ 核心检验通过 → 维持评级
  │   └─ 核心检验失败 → 降一级
  │
  └─ 综合逻辑链推断？
      └─ 标注 [INFERRED]
```

## 输出格式

在 reporting 阶段，解读文字中每条结论末尾标注标签：

```markdown
**结论1**：资产负债率对盈利能力有显著负向影响（β=-0.532, p<0.01）。[VERIFIED]

**结论2**：该效应在大型企业中更为明显。[INFERRED]（基于分组回归系数差异，但未做交互项正式检验）

**建议**：后续可考虑引入宏观经济变量考察周期效应。[SPECULATIVE]
```

## 与 reporting.md 的关系

- 本协议负责"结论可信度"维度
- reporting.md 负责"输出格式"维度
- 最终输出中，每个结论同时满足：①格式规范（reporting.md）+ ②可信度标签（本协议）
