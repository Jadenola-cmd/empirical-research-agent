# Panel Build Plan Protocol — 样本构建确认

## 位置

标准流水线 Step 4B。涉及多表合并、面板构建或回归样本筛选时必须通过本协议。

## 必填字段

多表数据回归前必须输出：

| 字段 | 要求 |
|---|---|
| 主表 | 样本基准表及其单位层级 |
| 合并表 | 每张被合并的数据表 |
| join key | 公司、年份、行业、地区等连接键 |
| join type | left / inner / outer 等 |
| 合并前样本量 | 主表及各合并表样本量 |
| 合并后样本量 | 每一步合并后的样本量 |
| 缺失值处理 | 删除、保留、填补或仅报告 |
| 缩尾处理 | 变量、分位点、执行时点 |
| 最终样本量 | 进入回归的观测数 |

## 阻断条件

出现以下任一情况，不得执行回归：

- 多表合并未说明主表。
- join key 或 join type 未明确。
- 合并前后样本量无法追踪。
- 缺失值处理会影响核心变量但未说明。
- 缩尾处理被执行但未披露。
- 用户尚未确认样本构建方案。

## 非交互降级

如果非交互环境无法等待用户确认，只能输出探索性报告，并在报告开头标注：

```text
[WARN] 未经过变量/样本确认，本报告仅为探索性结果，不得作为正式结论。
```
